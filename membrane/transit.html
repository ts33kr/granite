This method implements overridiable type installation mechanism similar to Beans in a way. It basically transfers remotable type (or a function) onto the remote site and aliases it under given token (name). The key here is this aliasing is done for every method transferred to the client side. And the definitions table that is used for aliasing is overridable based on inheritance. This allows you to override type definitions that may be used in the parent classes, without having to replace implementation code.
